# CalDrop

自由な書き方のテキストから予定を解析し、Googleカレンダーに一括登録できるWebツール。

## 使い方

1. テキストエリアに予定を入力（1行1予定）
2. 「解析する」をクリック
3. 解析結果を確認・必要なら編集
4. 登録したい予定にチェックを入れて「Googleカレンダーに登録」
5. 初回のみ Google 認証 → 以降は自動でスキップ

## 対応している入力形式

```
8月1日 10:00-15:00 散髪
8/3 15時から17時半 ジム
8/20-22 夏休み旅行
8月10日 休み
今日 15:00 歯医者
明日 10:00-11:00 打ち合わせ @渋谷オフィス
9/10 結婚式 会場：ホテルグランド （スピーチあり）
2025/4/5 14:00 過去の予定
```

| 形式 | 例 |
|------|----|
| 漢字日付 + コロン時刻 | `8月1日 10:00-15:00 散髪` |
| スラッシュ日付 | `8/3 15時から17時 ジム` |
| 「半」表記（=30分） | `8/3 14時半から16時 美容院` |
| 日付範囲（複数日） | `8/20-22 夏休み旅行` |
| 終日イベント | `8月10日 休み` |
| 相対日付 | `今日 15:00 歯医者` / `明日` / `明後日` |
| @場所名 | `会議 @渋谷オフィス` |
| 場所キーワード＋コロン | `会場：ホテルXX` / `場所：新宿` / `集合：渋谷駅` |
| 説明（丸括弧） | `（持ち物：PC・名刺）` |
| 過去の予定（年明示） | `2025/4/5 14:00 懇親会` / `2025年4月6日 10:00 面談` |

本日より前の日付は翌年に自動補正されます。過去の予定を登録したい場合は年を明記してください。
時刻の区切りは `-` `〜` `ー` `から` および空白など形式を問いません。
時刻表記は `15:00` `15時` `15時30分` `15時半` に対応しています。

## 特徴

- **完全フロントエンド** — サーバーなし・データ保存なし・プライバシー安全
- **無料で運用** — Vercel 無料プランでホスティング
- **認証の永続化** — Google 認証はトークンを localStorage に保存。再読み込み後も認証画面なし
- **一括登録** — チェックした予定をワンクリックで全件 Google カレンダーに登録
- **その場で編集** — 解析結果が間違っていたらカード上で直接修正
- **ソート** — 入力順 / 日付順を切り替え可能
- **予定の追加** — 解析後に「＋ 予定を追加」で手動追加可能
- **警告表示** — 認識できなかった行を一覧表示

## デザイン

「メモ帳に書く感覚」で使えるツールをコンセプトに、装飾を排除したミニマルな UI を目指しました。

### 参考にしたデザインシステム

- **[Notion](https://www.notion.so/)** — 暖色グレーのカラーパレットを参考。純白・純黒を避けた柔らかい印象。
- **[GitHub Primer](https://primer.style/)** — success / info / danger / warn の意味色設計を参考。
- **[Radix UI](https://www.radix-ui.com/colors)** — ダークモードの意味色トーンを参考。

### カラーパレット

| 用途 | ライト | ダーク |
|------|--------|--------|
| ページ背景 | `#f8f7f4`（暖色オフホワイト） | `#1b1b19` |
| カード・入力欄 | `#ffffff` | `#242420` |
| 本文テキスト | `#37352f`（Notion 準拠） | `#eeede6` |
| 補足テキスト | `#787774`（Notion 準拠） | `#8a8982` |
| ロゴの「.」 | `#2e9e5e`（グリーン） | 同左 |

ライト・ダークモード両対応（OS 設定に自動追従）。

## テスト

解析ロジック（`js/parser.js` / `js/gcal.js`）には Node.js 標準のテストランナーによるユニットテストがあります：

```bash
node --test
```

## セットアップ（ローカル開発）

### 1. Google Cloud Console の設定

1. [Google Cloud Console](https://console.cloud.google.com/) でプロジェクトを作成
2. 「APIとサービス」→「ライブラリ」で **Google Calendar API** を有効化
3. 「APIとサービス」→「OAuth 同意画面」を設定
4. 「認証情報」→「OAuth 2.0 クライアント ID」を作成
   - アプリケーションの種類：ウェブアプリケーション
   - 承認済み JavaScript 生成元：`http://127.0.0.1:5500`（ローカル）
5. 発行されたクライアント ID をコピー

### 2. 設定ファイルの作成

```bash
cp js/config.example.js js/config.js
```

`js/config.js` を開き、クライアント ID を貼り付ける：

```javascript
const GOOGLE_CLIENT_ID = 'あなたのクライアントID.apps.googleusercontent.com';
```

### 3. ローカルサーバーで開く

VS Code の Live Server（`http://127.0.0.1:5500`）などで `index.html` を開く。

## デプロイ（Vercel）

1. このリポジトリを GitHub に push
2. [Vercel](https://vercel.com/) でリポジトリを連携
3. Google Cloud Console の「承認済み JavaScript 生成元」に Vercel の URL を追加
