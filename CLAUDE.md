# CalDrop 開発ガイド

## プロジェクト概要
自由な書き方のテキストから予定を解析し、Googleカレンダーに一括登録できるWebツール。

## 開発者プロフィール
- Python 経験者。HTML / CSS / JavaScript はほぼ未経験
- 実装時は「なぜそう書くのか」「Python との違いは何か」など解説を挟みながら進める
- 専門用語には補足説明を付ける

## 基本ルール
- チャットは全て日本語で行うこと
- コードのコメントも日本語で書くこと
- 不明点は必ず確認してから実装すること
- 実装前に何をするか説明してから進めること
- コードの変更点は Python との比較や背景を交えて解説すること

## 技術スタック
- HTML / CSS / JavaScript（バニラJS）
- サーバーなし・APIなし・完全フロントエンドで動作
- ホスティング：Vercel（GitHub連携で自動デプロイ）

## テスト
- 解析ロジック（parser.js / gcal.js の純粋関数）は Node.js 標準テストランナーでテストする
- 実行：プロジェクト直下で `node --test`
- parser.js / gcal.js の末尾に `module.exports` ガードがあり、ブラウザ・Node 両対応
- **解析ロジックを変更したら必ずテストを実行し、新パターン追加時はテストも追加すること**

## 主な機能
- 日本語の自由なテキストから日付・時刻・タイトル・場所・説明を解析
- 複数予定の一括解析（1行1予定）
- チェックボックスで登録する予定を選択可能
- 「Googleカレンダーに登録」で全予定を一括登録（Google Calendar API 使用）
- 個別の「＋ 個別追加」リンクで1件ずつ URL 経由登録も可能
- 解析後のカードをその場で編集（タイトル・日付・時刻・場所・説明）
- 入力順 / 日付順のソート切り替え

## 対応している入力形式
```
8月1日 10:00-15:00 散髪
8/3 15時から17時 ジム
8/20-22 夏休み旅行
8月10日 休み
今日 15:00 歯医者
明日 10:00-11:00 打ち合わせ @渋谷オフィス
9/10 結婚式 会場：ホテルグランド （スピーチあり）
2025/4/5 14:00 過去の予定（年を明示）
```

### 日付指定
- `4月20日` / `4/20` — 漢字・スラッシュ両対応
- `4/25-27` — 日付範囲（複数日終日イベント）
- `今日` / `明日` / `明後日` — 相対日付（行の途中でも認識。「明日香」などの誤検出は否定先読みで回避）
- 本日より前の日付は自動的に翌年に補正
- `2025/4/5` / `2025年4月6日` — 年を明示すると補正をスキップ（過去の予定登録に使用）
- 実在しない日付（13月・45日）・時刻（25時）は弾く（その行は警告表示）

### 時刻の区切り文字
`-` `〜` `~` `～` `ー` `から` および半角・全角スペース すべて対応。
開始時刻のみ指定した場合、終了時刻は1時間後に自動設定（Googleカレンダーのデフォルトに準拠）。

### 場所の指定方法（3種類）
- `@会議室A` — @マークに続けて記述
- `場所：新宿オフィス` / `会場：ホテルXX` / `集合：渋谷駅` など — キーワード＋コロン
- `東京` `大阪` `京都` など主要都市名 — 単語単独でも検出（フォールバック）

### 説明の指定方法
- `（持ち物：PC・名刺）` / `(事前にレポート提出)` — 丸括弧内のテキスト

## デザイン思想

### コンセプト
「メモ帳に書く感覚」で使えるツール。装飾を排除し、入力とアクションに集中できるミニマルな UI を目指した。

### 参考にしたデザインシステム
- **[Notion](https://www.notion.so/)** — テキストカラー（`#37352f`）・サブテキスト（`#787774`）・背景（`#f8f7f4`）の暖色グレー体系を参考。純白・純黒を避けた柔らかさが特徴。
- **[GitHub Primer](https://primer.style/)** — 意味色（success / info / danger / warn）の色設計を参考。
- **[Radix UI](https://www.radix-ui.com/colors)** — ダークモードの意味色トーンを参考。

### カラーパレット

| 変数 | ライト | ダーク | 用途 |
|------|--------|--------|------|
| `--bg` | `#f8f7f4` | `#1b1b19` | ページ背景 |
| `--surface` | `#ffffff` | `#242420` | カード・入力欄 |
| `--border` | `#e6e5df` | `#383834` | 枠線 |
| `--text` | `#37352f` | `#eeede6` | 本文 |
| `--text-sub` | `#787774` | `#8a8982` | 補足・ラベル |
| `--accent` | `#37352f` | `#eeede6` | プライマリボタン背景 |

意味色（success / info / danger / warn）は背景色と文字色のペアで定義し、ライト・ダーク両対応。

### ロゴ
- `CalDrop` テキスト + グリーンの `.`（`#2e9e5e`）のみ
- 「.」は「ドロップ（Drop）」と「入力の手軽さ（メモの終止符）」を表現

### フォント
- UI 全体：システムフォント（`-apple-system`, `Hiragino Sans`, `Yu Gothic`）
- テキストエリア・コードブロック：等幅フォント（`SF Mono`, `Menlo`, `Consolas`）

## Google Calendar API 設定
- Google Cloud Console でプロジェクト作成済み
- OAuth クライアントID を `js/config.js` に記載
- config.js はフロントエンドのみで使用し、承認済み JavaScript 生成元（オリジン）による保護があるため Git 管理内で問題なし
- 承認済み JavaScript 生成元：`http://127.0.0.1:5500`（ローカル）、Vercel URL（本番）
- アプリは「テスト」モード。他ユーザーに公開するときのみ「本番環境」への移行が必要
- テストユーザー：登録済み（非公開）

## 認証の仕組み
- Google Identity Services（GIS）の OAuth トークンモデルを使用
- アクセストークンは1時間で失効（Google の仕様、変更不可）
- 2回目以降は `prompt: 'none'` でサイレント取得（ページ再読み込み後も認証画面なし）
- `localStorage` に認証済みフラグを保存。Googleにログイン中の限り永続的に動作
- スマホで `prompt:'none'` が応答を返さない場合、4秒タイムアウトで通常フローにフォールバック

## バージョン管理
- バージョン形式：`v0.x.x`（セマンティックバージョニング）
